# 文档模板和编写规范

本文档定义了学习计划中所有文档的标准模板和编写规范，确保内容的一致性和可读性。

## 📝 通用编写规范

### 文件命名规范

- 使用小写字母和连字符：`file-name.md`
- README 文件使用大写：`README.md`
- 项目文件夹使用小写和连字符：`project-name/`

### Markdown 格式规范

**标题层级：**
```markdown
# 一级标题（文档标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（详细说明）
```

**代码块：**
```markdown
\`\`\`javascript
// JavaScript 代码
const greeting = 'Hello, World!';
console.log(greeting);
\`\`\`

\`\`\`typescript
// TypeScript 代码
const greeting: string = 'Hello, World!';
console.log(greeting);
\`\`\`
```

**强调和标记：**
- **粗体**：`**重要内容**`
- *斜体*：`*强调内容*`
- `代码`：`` `code` ``
- ~~删除线~~：`~~错误内容~~`

**列表：**
```markdown
- 无序列表项 1
- 无序列表项 2
  - 嵌套列表项

1. 有序列表项 1
2. 有序列表项 2
```

**链接：**
```markdown
[链接文本](./path/to/file.md)
[外部链接](https://example.com)
```

**图片：**
```markdown
![图片描述](./images/image.png)
```

### 代码注释规范

**JavaScript/TypeScript 注释：**
```javascript
// ✅ 推荐：清晰的单行注释
const PI = 3.14159;

// ❌ 避免：不清晰的注释
const x = 5;

/**
 * 💡 提示：多行注释用于函数说明
 * @param {string} name - 用户名称
 * @returns {string} 问候语
 */
function greet(name) {
  return `Hello, ${name}!`;
}

// 🔍 深入理解：这里解释复杂的概念
// 闭包允许函数访问外部作用域的变量

// ⚠️ 警告：这是常见的陷阱
// 不要在循环中使用 var
```

**符号说明：**
- ✅ 推荐的做法
- ❌ 应该避免的做法
- 💡 提示和注意事项
- 🔍 深入理解
- ⚠️ 警告和常见陷阱
